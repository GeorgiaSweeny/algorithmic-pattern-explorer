# Stakeholder consultation - Educator Feedback Summary & Design Implications

Early stakeholder consultation was conducted with five educators from primary, secondary, further education, and workplace training backgrounds. Participants reviewed initial design concepts, interface sketches, and proposed functionality, providing formative feedback that informed the educational requirements, interaction design, and learning objectives of the application.

## 1. Overview

Feedback was gathered from **five educators** with experience across:

- Primary education  
- Secondary education  
- College / further education  
- Workplace training / professional development  

Participants were shown:

- early concept sketches  
- system layout (node graph, canvas, documentation panel)  
- proposed interaction model (step-through algorithm exploration)  

They were asked to evaluate:

- educational value  
- clarity of concept  
- suitability across learner levels  
- potential classroom integration  
- missing features or risks  

---

## 2. High-Level Validation

### 2.1 Positive Findings

Across all participants, there was consensus that:

- The concept has clear educational potential  
- Visualising algorithms as interactive structures is valuable  
- The combination of:
  - node graph (structure)
  - canvas (output)
  - documentation (explanation)

  is pedagogically strong  

### 2.2 Core Strength Identified

The most consistently valued feature was:

> the ability to step through an algorithm and observe how each computational operation changes the output

This supports:

- procedural understanding  
- cause-and-effect reasoning  
- decomposition of complex systems  

---

## 3. Key Questions Raised by Educators

### 3.1 Target Audience Definition

A recurring concern was:

> Who is this for?

Suggested ranges:

- late primary  
- secondary  
- college-level learners  
- adult self-directed learners  

**Implication:**  
The system must support multiple learning levels.

---

### 3.2 Educational Depth vs Tool vs Curriculum

Educators asked whether the system is:

- standalone learning tool  
- classroom teaching resource  
- or structured curriculum system  

**Implication:**  
Must support:
- self-directed learning  
- structured lesson modes  
- educator-led use (future)

---

### 3.3 Need for Multi-Level Explanation

Recommendation:

> explanations should adapt to prior knowledge

Levels:

- beginner (plain language)
- intermediate (conceptual)
- advanced (technical / mathematical)

**Implication:**  
Node documentation must support layered explanation.

---

### 3.4 Multi-Modal Learning Importance

Strong support for combining:

- visual demonstration  
- written explanation  
- interaction  
- repetition  

**Implication:**  
Each node should include:

- text explanation  
- visual behaviour  
- parameter interaction  
- optional animation  

---

### 3.5 Guided Onboarding

Recommendations:

- tutorials for first-time users  
- tooltips and hover explanations  
- guided prompts  

**Implication:**  
A progressive onboarding system is required.

---

### 3.6 Clear Learning Outcomes

Learners need:

- clear goals  
- explicit purpose  
- understanding of why tasks matter  

**Implication:**  
Nodes must explicitly map to:

- computational thinking concepts  
- learning objectives  
- progression structure  

---

### 3.7 Engagement & Motivation

Suggestions:

- gamification  
- achievements  
- challenges  
- unlockable content  
- progression systems  

**Implication:**  
Architecture should allow future gamification layers.

---

### 3.8 Assessment Ideas

Proposed tasks:

- identify node function from animation  
- rebuild patterns using nodes  
- debug incorrect graphs  
- match node graph to output  
- explain computational effect  
- constraint-based challenges  

**Implication:**  
Potential future assessment engine using node metadata.

---

### 3.9 Cognitive Load & Clarity

Concern:

> too much information can overwhelm learners

Recommendations:

- chunk information into small sections  
- progressive disclosure  
- contextual help  
- minimal initial UI  

**Implication:**  
Supports tier system + step-through design.

---

### 3.10 Progression of Difficulty

Strong support for:

- simple → complex progression  
- gradual introduction of concepts  

**Implication:**  
Validates tier system:

- core  
- intermediate  
- advanced  

---

### 3.11 Concept Linking Across Algorithms

Key insight:

> learners should recognise shared concepts across systems

Example:

- rotation in multiple algorithms  
- repetition in multiple systems  
- randomness across contexts  

**Implication:**  
Supports reusable node model and cross-algorithm learning.

---

### 3.12 System Scope Questions

Questions raised:

- Will code be visible?  
- Can users create their own patterns?  
- Can content be saved/shared?  

Potential modes:

- Explorer Mode  
- Sandbox Mode  
- Builder Mode (future)  
- Code View (optional future)  

---

## 4. Design Implications

### 4.1 Must-Have (MVP)

- Node-based exploration  
- Step-through execution  
- Canvas output  
- Documentation panel  
- Parameter editing  
- Tiered algorithm complexity  

---

### 4.2 Should-Have

- onboarding tutorial  
- tooltips / contextual help  
- multi-level explanations  
- algorithm comparison view  
- concept highlighting  

---

### 4.3 Future Enhancements

- gamification system  
- quizzes and assessments  
- saving/sharing patterns  
- curriculum mode for educators  
- optional code-view layer  

---

## 5. Evaluation Relevance

This feedback informs dissertation evaluation criteria:

- usability across age groups  
- learning effectiveness  
- transfer of knowledge between algorithms  
- cognitive load reduction  
- engagement and motivation  

---

## 6. Key Insight

> The system is most valuable when it supports structured learning progression, not just visual exploration.

This reinforces:

- node-based abstraction  
- tiered learning model  
- step-through visualisation  
- multi-modal explanation system  

---

## 7. Summary

Educator feedback strongly supports the system direction while emphasising:

- multi-level learning support  
- structured progression  
- explicit learning outcomes  
- reduced cognitive load  
- interactive learning design  
- transferable computational thinking concepts  

This confirms the project as:

> a structured educational framework for computational thinking through generative systems
